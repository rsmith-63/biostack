# Project Context: Bio-Research React-Koa Scaffold

## 1. Technical Stack & Environment
- **Runtime:** Node.js (Latest stable)
- **Frontend:** React 19.2.4 + Vite (Strictly use React 19 features like `use`, Action transitions, and the new Metadata API).
- **Backend:** Koa.js with Middleware-based architecture.
- **Hardware Target:** Apple Silicon (M4, arm64). Ensure all binaries/containers are native arm64.
- **Shell:** Zsh (All CLI commands must be Zsh-compatible).

## 2. Configuration Standards
- **Environment:** Always prioritize variables from the project-root `.env` file. 
- **Dependencies:** Use `npm` for package management. 
- **MCP Servers:** This project utilizes `@iflow-mcp/pubmed-mcp-server` for genomics and PubMed data retrieval.

## 3. Coding Guidelines
- **React:** - Favor Functional Components over Class Components.
  - Use the new React 19 `use()` hook for handling promises and context where appropriate.
  - Avoid deprecated lifecycle methods.
- **Koa:**
  - Use `async/await` for all middleware.
  - Keep routes modular in a `/routes` directory.
- **Architecture:** Maintain a clear separation between the Bio-Research logic (fetching from NCBI) and the UI layer.

## 4. Operational Instructions
- When creating new files, ensure they follow the existing folder structure: `/src/client` for React and `/src/server` for Koa.
- If an `npm` install is required for a new feature, proceed with the install and then update the `package.json` accordingly.
- Use the PubMed MCP tool whenever the user asks for scientific data or paper summaries.
